# Hypagraph v0.6

Hypagraph v0.6 adds trusted evaluation contracts and the first complete root Hypagoal controller for Pi.

## Root Hypagoal

Start one durable objective from ordinary prose:

```text
/hypagoal Implement the requested repository change and prove it with the existing checks.
```

Hypagraph authors one canonical workflow, preserves the objective exactly, and continues through runnable tasks, checks, gates, disconnected branches, and bounded iteration regions.

Completion remains workflow-derived. The model cannot mark a goal complete.

## Controls

```text
/hypagoal status
/hypagoal pause [reason]
/hypagoal resume
/hypagoal cancel [reason]
/hypagoal graph
```

Status and graph views show the objective, workflow phase, goal state, active and next work, ready work, remaining turn and token budgets, loop and evaluation state, canonical blockage, automatic revision state, and explicit stop reason.

## Trusted evaluation

v0.6 separates:

- evaluation validity;
- numeric progress;
- typed success;
- development, probe, and holdout purpose;
- transparent, protected, and future isolated trust;
- evaluator integrity and feedback policy;
- loop limits and evaluation budgets.

Invalid observations do not update accepted progress or satisfy success.

## Safe autonomous control

The controller provides:

- durable state-bound continuation identity;
- deterministic fairness across independent components;
- exactly-once turn and token accounting;
- explicit reload and branch-change pause;
- deterministic budget termination;
- one bounded automatic workflow revision for recoverable repository blockers;
- non-weakening validation for evidence, checks, gates, typed success, evaluation trust, limits, scopes, dependencies, and existing required work;
- replay and restore without semantic dispatch.

## Install

```bash
pi install git:github.com/Hypabolic/Hypagraph
```

Update an existing installation:

```bash
pi update git:github.com/Hypabolic/Hypagraph
```

## Deferred

v0.6 remains root-only and sequential in one Pi session. Goal families, child Hypagoals, isolated executors, worktree integration, bounded physical concurrency, ACP, and named direct agent adapters remain planned later milestones.
